    Binary Classification API

    This page documents the main entry points for binary classification experiments in Batwing-ML.

    • Optuna-based hyperparameter tuning
    • Nested cross-validation for unbiased model comparison
    • Rich evaluation utilities (metrics + plots)

    If mkdocstrings fails to import something, adjust the dotted paths below to match your actual module layout.

    Hyperparameter tuning (binary classification)

    from batwing_ml import hyperparameter_tuning_classification

    hyperparameter_tuning_classification(X=None, y=None, model_class=None, param_grid=None, scoring='accuracy', n_trials=50, cv_folds=5, stratified=True, direction='maximize', verbose=True, return_model=True, random_state=42, use_fraction=None, use_n_samples=None, fast_mode=False)

    üîß Hyperparameter Tuning for Classification (via Optuna)

    Optimize hyperparameters of any binary classification model using Optuna‚Äôs efficient sampling. This function supports cross-validation, custom scoring, stratified sampling, reproducibility, and returns the best trial summary and optionally the best fitted model.

    Ideal for automated model selection pipelines, leaderboard tuning, or experimentation in research and production ML workflows.

    Parameters

    X : pd.DataFrame or np.ndarray Feature matrix with shape (n_samples, n_features).

    pd.Series or np.ndarray

    Target vector with binary labels (0 or 1).

    callable

    A scikit-learn-style classifier class (e.g., RandomForestClassifier, LogisticRegression). Not an instance ‚Äì must be the class itself.

    dict

    Dictionary mapping hyperparameter names to Optuna sampling functions. Example: { "C": lambda t: t.suggest_float("C", 0.01, 10, log=True), "penalty": lambda t: t.suggest_categorical("penalty", ["l1", "l2"]) }

    str or callable, default='accuracy'

    Metric to optimize during cross-validation. Supports: - Any string from sklearn (e.g., 'f1', 'roc_auc', 'log_loss') - A custom callable with signature: scorer(estimator, X_val, y_val)

    int, default=50

    Number of optimization trials to run.

    int, default=5

    Number of folds in cross-validation to evaluate each hyperparameter setting.

    bool, default=True

    If True, use StratifiedKFold for cross-validation (preserves class balance).

    {'maximize', 'minimize'}, default='maximize'

    Whether to maximize or minimize the scoring function.

    bool, default=True

    If True, prints trial progress, parameter table, and best result summary.

    bool, default=True

    If True, fits and returns the best model using the entire dataset.

    int, default=42

    Random seed for reproducible folds and results.

    float or None, optional

    If provided, samples a random fraction (e.g., 0.1 = 10%) of the dataset.

    int or None, optional

    If provided, uses only the first N rows of the data.

    bool, default=False

    If True, reduces n_trials to 10, disables logs, and optimizes speed. Use for quick tests or large-scale experiments.

    Returns

    dict A dictionary containing: - 'best_score' : float - 'best_params' : dict - 'study' : optuna.Study - 'best_model' : fitted model (if return_model=True)

    Raises

    ValueError If required arguments are missing or incompatible.

    User Guide

    üß† When to Use: - You're comparing models or trying to find optimal settings for one. - You want to replace GridSearchCV with faster, smarter search. - You want control over cross-validation, scoring, and sampling. - You‚Äôre tuning on large datasets and want quick feedback via fast_mode or subsampling.

    üìå Core Concepts:

    ‚Ä¢ Optuna Trials: Each trial samples a different set of parameters using Optuna‚Äôs intelligent search strategy (Tree Parzen Estimator) and evaluates them using cross-validation.

    ‚Ä¢ param_grid: Define hyperparameter ranges via lambdas ‚Äî much more flexible than grid search. Supports suggest_float, suggest_int, suggest_categorical, and log-scale sampling.

    ‚Ä¢ scoring: Choose a string (e.g., 'f1', 'accuracy', 'roc_auc') or define a custom scoring function that returns a float. Examples: - 'neg_log_loss': minimizes log loss (automatically handled) - custom_cost(estimator, X, y): returns cost/loss based on predictions

    ‚Ä¢ StratifiedKFold: Recommended for binary classification, especially if classes are imbalanced.

    ‚Ä¢ fast_mode: Use fast_mode=True to reduce trials to 10, turn off prints, and return results quickly. Ideal for initial tests or iterative tuning on large data.

    ‚Ä¢ Sampling Subsets: - use_fraction=0.1: randomly sample 10% of the data - use_n_samples=5000: use only the first 5000 rows - You can pass both; use_n_samples is applied after use_fraction

    üß™ Example Usage:

    from sklearn.ensemble import RandomForestClassifier results = hyperparameter_tuning_classification( X=X, y=y, model_class=RandomForestClassifier, param_grid={ "n_estimators": lambda t: t.suggest_int("n_estimators", 50, 300), "max_depth": lambda t: t.suggest_int("max_depth", 3, 15), "min_samples_split": lambda t: t.suggest_int("min_samples_split", 2, 10), }, scoring='f1', n_trials=50, stratified=True, fast_mode=False, return_model=True )

    print(results["best_params"]) print(results["best_score"]) best_model = results["best_model"]

    üí° Tips: - Use fewer trials with small data or fast models (e.g., LogisticRegression). - Tune for log_loss when probabilistic accuracy matters (e.g., fraud detection). - Always use return_model=True in production workflows to get the fitted model. - Set fast_mode=True when running in a loop or with large datasets. - Pair with evaluate_classification_model() for post-tuning performance analysis.

    See Also

    evaluate_classification_model : Full evaluation suite for binary classifiers. GridSearchCV : Traditional brute-force alternative (slower, less efficient). Optuna : https://optuna.org

    Nested cross-validation (binary classification)

    from batwing_ml import run_nested_cv_classification

    Run nested cross-validation for multiple classification models with optional tuning strategies and scaling.

    This function evaluates one or more classification models using nested cross-validation ‚Äî an approach that combines robust performance estimation with internal hyperparameter tuning. It supports both GridSearchCV and RandomizedSearchCV, allows sub-sampling for large datasets, and offers a simplified or silent fast mode for production or large-scale runs.

    Parameters

    X : pd.DataFrame or np.ndarray Feature matrix (n_samples, n_features). y : pd.Series or np.ndarray Binary classification target variable (n_samples,).

    dict of str ‚Üí estimator

    Dictionary mapping model names to scikit-learn classifiers.

    dict of str ‚Üí dict

    Dictionary mapping model names to hyperparameter grids in sklearn format.

    list of str or callables, default=['accuracy']

    List of scoring metrics to evaluate during nested CV (e.g., ['accuracy', 'roc_auc']).

    int, default=3

    Number of folds in the outer CV loop (used for final performance estimation).

    int, default=3

    Number of folds in the inner CV loop (used for hyperparameter tuning).

    {'grid', 'random'}, default='grid'

    Search strategy to use for tuning: - 'grid': exhaustive parameter search (GridSearchCV) - 'random': random search with n_iter (RandomizedSearchCV)

    int, optional

    Only used when search_method='random'. Number of parameter settings sampled.

    int, optional

    If provided, limits the number of rows used for fitting to this count (e.g., 100_000).

    float, optional

    If provided, randomly samples this fraction of rows (e.g., 0.1 for 10%).

    callable, default=lambda name: name not in ['random_forest', 'gboost']

    Function to determine if a given model should use StandardScaler.

    sklearn-compatible transformer, optional

    If provided, replaces built-in scaling logic with a user-defined transformer or pipeline.

    bool, default=True

    If True, prints progress, hyperparameter tables, and detailed summaries.

    bool, default=False

    If True, disables visual output and reduces outer CV folds to speed up runtime.

    bool, default=False

    If True, returns a dictionary with summaries, best parameters, and matplotlib figures.

    {'tabulate', 'rich'}, default='tabulate'

    Controls how summary tables are printed (colorful or ASCII).

    Returns

    dict, optional Only if return_results=True. Includes: - 'summary': pd.DataFrame of all model-metric combinations - 'results': detailed CV results and best parameters - 'best_params': dict of best params for each model and metric - 'figures': optional comparison plots (if fast_mode=False)

    Raises

    ValueError If any of the required inputs (X, y, model_dict, param_grids) are missing.

    Examples

    from sklearn.linear_model import LogisticRegression from sklearn.ensemble import RandomForestClassifier

    models = { 'logistic': LogisticRegression(), 'rf': RandomForestClassifier() }

    grids = { 'logistic': {'C': [0.1, 1, 10]}, 'rf': {'n_estimators': [50, 100]} }

    results = run_nested_cv_classification( X=X, y=y, model_dict=models, param_grids=grids, scoring_list=['accuracy', 'f1'], fast_mode=True, search_method='random', n_iter=5, max_samples=10000, return_results=True )

    print(results['summary'])

    User Guide

    üß™ What is Nested CV? - Nested cross-validation helps prevent overfitting in model selection by using an inner loop for hyperparameter tuning and an outer loop for estimating generalization performance.

    üìå Common Use Cases: - Benchmarking many classifiers fairly. - Avoiding bias from tuning and testing on the same split. - Handling imbalanced or high-cardinality feature spaces.

    ‚öôÔ∏è When to Use Which Search Strategy: - Use 'grid' for small hyperparameter spaces where you want exact control. - Use 'random' when the space is large or you're optimizing speed vs. accuracy.

    ‚ö° When to Use fast_mode=True: - During large-scale experiments (e.g., 10M+ rows). - If visual plots or verbose logging are unnecessary. - In production or headless environments.

    üîç Interpreting the Output: - Mean/Std Dev: Represents model stability across folds. - Best Params: Optimal hyperparameters found by inner CV. - Figures: Comparison plots for accuracy, F1, AUC, etc.

    üìâ Scaling Rules: - By default, tree-based models skip scaling. - You can override this using use_scaling() or provide a custom preprocessor.

    üö´ Memory Tips for Big Data: - Use max_samples or sample_frac to downsample safely. - Enable fast_mode to avoid expensive operations like seaborn/Matplotlib rendering.

    See Also

    • GridSearchCV, RandomizedSearchCV (from sklearn)
    • evaluate_classification_model() for post-training model diagnostics

    Evaluation utilities (binary classification)

    from batwing_ml import evaluate_classification_model

    evaluate_classification_model(model=None, X_train=None, y_train=None, X_test=None, y_test=None, cv=5, cost_fn=None, cost_fp=None, validation_params=None, scoring_curve='accuracy', verbose=True, return_dict=False, return_model_only=False, export_model=False, extra_plots=None, sample_fraction=None, sample_size=None, fast_mode=False)

    Evaluate a binary classification model with diagnostics, metrics, plots, and model handling.

    This function evaluates a scikit-learn-compatible classifier using standard and advanced metrics. It supports both pretrained and unfitted models, can train during evaluation, and provides plots for calibration, threshold tuning, learning/validation curves, and more. It's useful for audits, experimentation, and production diagnostics.

    Parameters

    model : estimator object, optional A scikit-learn classifier. If unfitted, the function will train it using provided data.

    array-like, optional

    Training feature matrix. Required if model is not already fitted.

    array-like, optional

    Training labels.

    array-like, optional

    Testing feature matrix.

    array-like, optional

    Testing labels.

    int, default=5

    Number of folds for cross-validation in learning/validation curves.

    float, optional

    Cost for false negatives in misclassification cost calculation.

    float, optional

    Cost for false positives.

    dict, optional

    Dictionary of hyperparameter names to lists of values for validation curve plotting.

    str, default='accuracy'

    Scoring metric used in learning and validation curves.

    bool, default=True

    If True, prints metrics and plots.

    bool, default=False

    If True, returns metrics as a dictionary.

    bool, default=False

    If True, returns the trained model only (no metrics).

    bool, default=False

    If True, returns (metrics_dict, trained_model) tuple.

    list of str, optional

    Options include: - "threshold": Precision/Recall/F1 vs threshold - "calibration": Reliability of predicted probabilities - "ks": KS separation statistic - "lift": Lift curve - "det": Detection Error Tradeoff curve

    float, optional

    Fraction of data to use for train/test sets (e.g., 0.1 = 10%).

    int, optional

    Number of rows to use from training and testing sets.

    bool, default=False

    If True, skips plots and printouts for faster processing.

    Returns

    dict, estimator, or tuple - Dictionary of metrics if return_dict=True - Fitted model if return_model_only=True - (metrics_dict, model) if export_model=True

    Raises

    ValueError If necessary inputs are missing or misaligned.

    Examples

    evaluate_classification_model( model=clf, X_train=X_train, y_train=y_train, X_test=X_test, y_test=y_test, cost_fn=10, cost_fp=1, validation_params={'max_depth': [2, 4, 6]}, scoring_curve='f1', extra_plots=['threshold', 'ks'], return_dict=True )

    Save a newly trained model

    model = evaluate_classification_model( model=RandomForestClassifier(), X_train=X, y_train=y, X_test=Xt, y_test=yt, return_model_only=True )

    Evaluate a pretrained model

    from joblib import load model = load("model.joblib") evaluate_classification_model(model=model, X_test=Xt, y_test=yt)

    User Guide

    üß† When to Use This Function: ‚Ä¢ You have a binary classifier (trained or not) and want a full evaluation report. ‚Ä¢ You want to generate visual diagnostics to understand performance beyond raw metrics. ‚Ä¢ You're comparing different models and need standardized summaries and visual feedback. ‚Ä¢ You want to include cost-sensitive evaluation logic (e.g., FP/FN tradeoffs in fraud, medical diagnosis). ‚Ä¢ You plan to export a trained model right after evaluation to reuse or save for deployment.

    üìä Core Metrics Explained: ‚Ä¢ Accuracy: ‚Üí Proportion of correct predictions overall. ‚Üí Works well when classes are balanced, but misleading with imbalanced classes.

    ‚Ä¢ Precision:
    ‚Üí Of all predicted positives, how many were actually correct?
    ‚Üí Use when false positives are costly (e.g., spam detection).

‚Ä¢ Recall:
    ‚Üí Of all actual positives, how many did the model catch?
    ‚Üí Use when false negatives are costly (e.g., cancer diagnosis).

‚Ä¢ F1 Score:
    ‚Üí Harmonic mean of Precision and Recall.
    ‚Üí Best for imbalanced datasets where both FP and FN matter.

‚Ä¢ ROC AUC:
    ‚Üí Measures the model's ability to rank positives over negatives.
    ‚Üí Robust against imbalance. Closer to 1 = better.

‚Ä¢ Cost-sensitive Average Loss:
    ‚Üí Weighted loss calculation based on your domain-specific cost of False Positives and False Negatives.
    ‚Üí Useful in fraud detection, churn prediction, or medical triage where not all errors are equal.

    üìà Optional Diagnostic Plots: These are generated when extra_plots is set and fast_mode=False.

    ‚Ä¢ Threshold Curve (`extra_plots=['threshold']`):
    ‚Üí Shows how Precision, Recall, and F1 change with different probability thresholds.
    ‚Üí Use it when you need to manually tune the decision boundary (e.g., prioritize recall over precision).

‚Ä¢ Calibration Curve (`extra_plots=['calibration']`):
    ‚Üí Compares predicted probabilities to observed outcomes.
    ‚Üí Helps determine if model outputs represent true probabilities (e.g., in credit scoring, risk modeling).

‚Ä¢ KS Statistic (`extra_plots=['ks']`):
    ‚Üí Plots the cumulative distribution of scores for each class and measures the maximum separation.
    ‚Üí A higher KS value (closer to 1) indicates good class separation.

‚Ä¢ Lift Curve (`extra_plots=['lift']`):
    ‚Üí Compares model performance against random guessing in terms of capturing true positives.
    ‚Üí Great for targeting top decile groups (e.g., marketing response modeling).

‚Ä¢ DET Curve (`extra_plots=['det']`):
    ‚Üí Plots False Positive Rate vs. False Negative Rate using logarithmic scale.
    ‚Üí Especially useful in imbalanced classification (e.g., rare disease detection, security event modeling).

    üìö Learning & Validation Curves (CV Required): ‚Ä¢ Learning Curve: ‚Üí Plots model performance vs. training size. ‚Üí Helps detect underfitting (low train/val scores) or overfitting (train ‚â´ val). ‚Üí Useful to decide if more data will help your model.

    ‚Ä¢ Validation Curve (`validation_params={'C': [...], 'max_depth': [...]}`):
    ‚Üí Shows how a single hyperparameter affects training and validation performance.
    ‚Üí Use to find optimal model complexity (e.g., tree depth, regularization).

    ‚öô Runtime & Usability Tips: ‚Ä¢ fast_mode=True: ‚Üí Skip all visual output and verbose logs. Ideal for CI/CD pipelines or large batch runs.

    ‚Ä¢ return_model_only=True:
    ‚Üí Use this if your model is not yet fitted and you want the trained object back after evaluation.

‚Ä¢ export_model=True:
    ‚Üí Returns both (metrics_dict, trained_model) to chain into pipelines, dashboards, or export routines.

‚Ä¢ sample_fraction / sample_size:
    ‚Üí Quickly prototype or test on large datasets without full evaluation time/cost.
    ‚Üí Useful when training/testing on full 10M+ rows is not practical.
